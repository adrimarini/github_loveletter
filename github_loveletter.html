GitHub with the Terminal


Step 1 to loving you: Create a new repository on GitHub

Step 2: Open the terminal on your Mac

Step 3: Change the current working directory to your local project

Step 4: Initialize the local directory as a Git repo using commmand =$ git init

Step 5: Add the file(s) to your new local repository- this gets them staged for their first commit. Use command = $ git add .

Step 6: Commit the files that you have staged in your repo. Use command= $ git commit -m "first commit"

Step 7: At the top of your GitHub repo Quick set-up page, copy and paste the remote repository URL.

Step 8: In the terminal, paste the URL for the remote repository. $ git add origin url-goes-here then $ git remote -v

Step 9: Push the changes in your local repository to GitHub. Use command= $ git push original master


GitHub on the Website


To create a new repo on the GitHub website, click the + in the upper right corner next to your user name and click New Repository

Next, name your repository

Then, write a brief description

Lastly, select initialize this repository with a README, then click Create repository

To create a new Branch, go to your new repository.

Click the drop down at the top of the file list that says branch: master

Type a branch name into the new branch text box (this will be the name of the new branch)

Select the blue Create branch box or hit "enter" on the keyboard.

And wellah! You now have two new branches!

To make or commit a change, click the README.md file.

Click the pencil icon in the upper right corner to edit

Jot down some notes in the editor

Write a commit message that describes the changes you made

Click Commit changes


Next, we can make a Pull request or Merge a Pull Request

On your repo, there is a Pull Request tab. Click the tab, then click the green New pull request button.

select the branch.

Read over the changes in the diffs and make sure they are what you want to submit.

Then, click the big green Create pull request button.

Give it a title and add a breif description of the changes made.

And wellah you have created a Pull request!

To then Merge your Pull request, begin by clicking the green "merge pull request button".

Click confirm merge

Once you see the purple button on the tab saying you have succesfully merged and closed (wooh!), you can click the grey Delete branch button.

But enough small talk...
